Autonomous Intelligence Engines

Self-monitoring systems that proactively detect risks, generate insights, and plan corrective actions — no user intervention required.

Design Philosophy
These engines implement autonomous monitoring — they continuously analyze behavioral data, classify states into lifecycle phases, detect anomalies via statistical methods (linear regression, compound interest models, Mahalanobis distance), and generate actionable recommendations. Each engine operates independently but shares cross-cutting signals when metrics overlap.

Overview

The autonomous intelligence layer contains 9 engines that model different aspects of cognitive and behavioral health. Unlike standard CRUD services, these engines maintain rolling statistical windows, classify lifecycle phases, predict future states, and generate proactive interventions.

🧠 AttentionDebtService

lib/core/services/attention_debt_service.dart

Models cognitive overhead as financial debt — unresolved items accrue cognitive interest over time using compound interest formulas. When total debt exceeds capacity, the system declares cognitive bankruptcy and triggers emergency triage.

Core Concepts

  • Debt Item — any deferred decision, postponed task, or unresolved mental load
  • Cognitive Interest — baseCost × (1 + dailyRate)^daysOpen — grows exponentially
  • Postponement Penalty — each postponement adds 20% multiplier to effective cost
  • Debt Score — 0–100 scale mapped from total debt vs. 500-unit capacity via sigmoid
  • Bankruptcy Threshold — score ≥ 80 triggers emergency triage mode

Debt Categories & Interest Rates

CategoryDaily InterestBase CostRationale
🙈 Avoided Conversation15%12Avoidance is the most expensive debt
⚔️ Unresolved Conflict12%16Conflicts compound emotional overhead fast
📨 Pending Response10%6Social obligations create ambient anxiety
🤷 Deferred Decision8%10Every open decision consumes working memory
📚 Information Backlog6%7Unprocessed info creates guilt loops
⏳ Postponed Task5%8Tasks grow harder the longer they wait
🎯 Undefined Goal4%9Vague goals drain background attention
🏗️ Incomplete Project3%14Slow burn but high base cost

Urgency Classification

Based on the ratio of effectiveCost / baseCost:

UrgencyRatio ThresholdMeaning
🟢 Low< 1.5×Recently added, manageable
🟡 Moderate1.5×–3×Starting to accumulate interest
🟠 High3×–6×Significant cognitive burden
🔴 Critical6×–12×Actively draining cognitive resources
💀 Bankruptcy≥ 12×Item alone is overwhelming

🚀 MomentumEngineService

lib/core/services/momentum_engine_service.dart

🐸 DriftDetectorService

lib/core/services/drift_detector_service.dart

The "boiling frog" detector — monitors gradual lifestyle regression across multiple metrics using linear regression over rolling windows. Detects slow negative changes that humans don't notice, predicts tipping points, and correlates cross-metric slides.

Drift Severity Levels

LevelMeaningAction
📈 ImprovingMetric trending positivelyReinforce current behavior
➡️ StableNo significant changeContinue monitoring
📉 DriftingSlight negative trend detectedAwareness nudge
⚠️ SlidingConsistent downward trajectoryActive intervention recommended
🆘 FreefallRapid degradationEmergency recovery protocol

How It Works

  1. Data Collection — aggregates daily metric values from other services (sleep quality, mood, productivity, etc.)
  2. Rolling Regression — fits linear regression over configurable windows to detect slope direction
  3. Tipping Point Prediction — extrapolates when a metric will cross a danger threshold at current rate
  4. Cross-Metric Correlation — detects when multiple metrics drift together (e.g., sleep down → mood down → productivity down)
  5. Recovery Nudges — generates targeted suggestions based on which metrics are sliding and their correlations

generateReport()DriftReportFull drift analysis with recovery suggestions

⚡ WillpowerBudgetService

lib/core/services/willpower_budget_service.dart

Models daily willpower as a finite, depletable resource based on ego depletion theory. Starts at 100 each day, drained by cognitive demands (decisions, resistance, focus blocks), and partially restored by strategic recovery actions. Predicts decision fatigue windows and rates how well the user manages cognitive resources.

Willpower Zones

ZoneBudget RangeBehavior
🟢 Full Tank80–100Peak cognitive capacity — make important decisions now
🟡 Comfortable60–80Good capacity — routine decisions and focused work
🟠 Stretching40–60Declining — avoid major decisions, use routines
🔴 Depleted20–40Decision fatigue active — autopilot mode recommended
💀 Empty0–20Ego depletion — rest immediately, no decisions

Demand & Recovery Model

  • Demands — decisions, resisting temptation, sustained focus, context switches, emotional regulation
  • Recovery — rest breaks, meals, exercise, meditation, nature exposure, sleep
  • Fatigue Forecast — predicts when budget will cross each zone boundary at current drain rate
  • Strategic Score — rates how well the user allocates high-demand tasks to high-budget periods

🔮 RegretMinimizationService

lib/core/services/regret_minimization_service.dart

Inspired by Bezos's regret minimization framework — records decisions with context and stakes, tracks outcomes over time, detects regret patterns (action vs. inaction), identifies recurring cognitive biases, and distills personalized decision-making wisdom. The ultimate question: "Will 80-year-old me regret NOT doing this?"

Decision Domains

Decisions are categorized across 8 life domains: Career, Relationships, Health, Finance, Education, Lifestyle, Creative, and Other. Each domain builds its own regret profile over time.

Analysis Pipeline

  1. Decision Recording — log decisions with context, stakes, emotional state, and alternatives considered
  2. Outcome Tracking — revisit decisions after configurable time periods to record actual outcomes
  3. Regret Classification — categorize outcomes as regret of action (did it, wish I hadn't) or regret of inaction (didn't do it, wish I had)
  4. Bias Detection — identify recurring cognitive biases from decision patterns (status quo bias, loss aversion, sunk cost, etc.)
  5. Wisdom Distillation — generate personalized decision-making principles from your history

Integration Pattern
The autonomous engines work best when connected to each other's output signals. For example, the Drift Detector can feed into Burnout Detector, the Momentum Engine can inform Streak Guardian risk assessment, and the Willpower Budget can influence Ritual Engine micro-adjustments.
